# Code of Conduct

## Our pledge

We are committed to making the Good First Issues community welcoming,
respectful, and inclusive for everyone.

Everyone who participates in this project should be able to contribute in good
faith, regardless of background, identity, experience level, or personal
circumstances.

## Expected behavior

Examples of behavior that helps sustain a healthy community include:

- Being respectful and considerate in issues, pull requests, reviews, and other
  project spaces.
- Welcoming newcomers and helping first-time contributors understand the
  project.
- Accepting constructive feedback gracefully.
- Giving feedback that is specific, kind, and focused on the work.
- Taking responsibility for mistakes and correcting them when needed.
- Respecting different viewpoints and experiences.

## Unacceptable behavior

Examples of unacceptable behavior include:

- Harassment, intimidation, threats, or sustained disruption.
- Insults, personal attacks, or demeaning comments.
- Discriminatory language or behavior.
- Unwelcome sexual attention or sexualized language.
- Publishing private information without permission.
- Any behavior that would reasonably make participation unsafe or unwelcome.

## Scope

This Code of Conduct applies in all project spaces, including GitHub issues,
pull requests, discussions, reviews, and any other space where someone is
representing the Good First Issues project.

## Reporting and enforcement

If you experience or observe unacceptable behavior, report it to the project
maintainers through the available GitHub project channels.

If a report contains private or sensitive information, do not post those details
publicly. Instead, contact a maintainer through GitHub and ask for an appropriate
private reporting channel.

Maintainers are expected to review reports in good faith and take action that is
appropriate to the situation. Possible actions may include clarification,
moderation of comments, temporary restrictions, or removal from project spaces.
